🚀 Publicar en Vercel (modo blindado)

Genera un index.html completamente sanitizado:

  • 🖼 Imágenes embebidas en base64 (sin URLs externas)
  • 🔒 Links de proveedor eliminados del HTML público
  • 💰 Costos CNY del proveedor ocultos
  • 🚫 Referencias textuales a la fuente reemplazadas

El archivo pesa ~1-2 MB con imágenes comprimidas. Es imposible que un cliente o IA detecte el origen.

Vaciar caché si las imágenes se ven mal o si el archivo está pesando demasiado (forzará recompresión).

⚡ Auto-deploy a Vercel (un click)

Configurá tu token una vez y después cada cambio se publica con un click. Sin descargar archivos, sin GitHub.

🔑 Conseguir token: vercel.com/account/settings/tokens → "Create" → ámbito "Full Account"
📦 Project ID: en tu dashboard de Vercel, Settings → General → "Project ID"

🔒 El token se guarda solo en tu navegador. NO se incluye nunca en el HTML público.

📦 GitHub sync (opcional, recomendado)

Si lo configurás, después de cada deploy a Vercel también se actualiza tu repo de GitHub. Ventajas:

  • 📜 Historial completo de cambios (cada deploy = un commit)
  • 🔄 GitHub y Vercel siempre sincronizados
  • ↩️ Rollback fácil a cualquier versión anterior
  • 🛡 Backup automático en cada cambio

🔑 Token: github.com/settings/tokens/new → ámbito "repo" → 90 días → Generate

🔒 El PAT se guarda solo en tu navegador. NO se incluye nunca en el HTML público.

💾 Backup y restauración

Tu admin guarda todo en el navegador. Si borrás caché, cambiás de compu o algo se rompe, podés restaurar desde un backup JSON.

Backup recomendado: cada vez que importes productos nuevos → "Exportar JSON" → guardar en Drive/Dropbox.
